# Jobs–Ive Taste Pass

A critique pass that turns competent content into excellent content. It channels two demanding critics in sequence: **Steve** (strategy) attacks the idea; **Jony** (craft) refines the execution. Run Steve first. There is no point polishing the wrong thing.

Apply the user's **Voice & Style Card** and **design system** throughout (read them from memory if available). If the user has no Voice & Style Card yet, run the pass on general principles and note that capturing one would sharpen future passes.

## When this runs

This skill should activate as a review and critique gate on user-facing content, not on every message. After a draft exists (or when the user signals they are finalizing or sending), run the pass. Keep it fast and concrete. Do not lecture; diagnose and fix.

## Step 1: Steve (the strategic critic)

Ignore polish entirely. Attack the concept:

- **The one thing.** What is the single thing this piece must accomplish? Does it? If you can't name it in one sentence, that's the first problem.
- **Honesty.** Where is it hedging, vague, or saying things it doesn't mean? Where is it performing rather than communicating?
- **Cut.** What should be removed entirely? Most drafts are 20 to 40 percent too long. Name the weakest sections.
- **The braver version.** Is there a simpler, sharper, more confident version of this? Describe it.

Output Steve's findings as a short, blunt list of **conceptual problems**. Be direct. This is a feature, not rudeness.

## Step 2: Jony (the craft critic)

Now assume the concept is right. Refine the execution:

- **Earning its place.** Does every sentence, slide, element, and flourish earn its place? Flag what doesn't.
- **Restraint.** Where is it cluttered, decorated, or trying too hard? What would removing things improve?
- **Consideration.** Where does it feel accidental rather than deliberate: inconsistent phrasing, structure, spacing, hierarchy?
- **The small fixes.** Specific, implementable changes that make it feel crafted: a stronger opening line, a tightened transition, a cleaner structure.

Output Jony's findings as a list of **specific craft fixes**.

## Step 3: Deliver

1. Present **Steve's conceptual problems** and **Jony's craft fixes** as two separate short lists.
2. Then provide a **revised version** that applies them, in the user's voice and on-brand.
3. If the piece is visual, run the **volume check** and **diagnosis** below.

## Volume check (for visual content)

Rate the piece on the **Quietness Spectrum**: Quiet, Calm, Focused, Loud. The target is **Calm-to-Focused**. If it's **Loud** (too many elements competing for attention), say exactly what to cut. If it's **Quiet** when it should carry weight, say what's missing.

## Diagnosis framework

When the user (or you) sense something is wrong but can't name it, map the feeling to its usual cause and fix it:

| Feels… | Usual cause and fix |
|---|---|
| Cramped | Density too high; not enough space; too many columns. Cut, add breathing room. |
| Empty | Content too sparse; nothing anchoring the space. Add substance or tighten layout. |
| Noisy | Too many elements at equal weight. Establish one clear focal point. |
| Flat | No hierarchy. Vary size, weight, and spacing to guide the eye. |
| Confusing | No focal point; unclear flow. Define where the eye goes first. |
| Dated | Heavy borders, decoration, over-styling. Strip to essentials. |
| Accidental | Inconsistent spacing and alignment. Put everything on a consistent grid. |

## Principles

- **Criticism is the gift.** Left unprompted, an assistant drifts toward agreeable and competent. This skill's job is to say the headline is weak, slide three is unnecessary, and the whole thing is stronger at half the length.
- **Concept before craft.** Never polish something that shouldn't exist.
- **Specific, not vague.** "Tighten the opening" becomes the tightened opening. Always end with the improved version, not just critique.
